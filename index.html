Dependencies & Risks

Planning & Alignment is key.

The path to green isn't just technical - it's orchestration. These are the dependencies we manage actively.

API Readiness

Need:Swagger + GitLab access, integration via TCS / P7
Risk:Delays block agent action capability
Fix:Prioritise API enablement for Tier 1 intents

Data & Knowledge Quality

Need:Accurate billing data, current product info
Risk:Wrong responses erode customer trust
Fix:Governance model for knowledge updates

Organisational Alignment

Need:VMO2, Google, Accenture in lockstep
Risk:Delivery friction across stakeholders
Fix:Clear ownership model - defined per stage
Trust, Risk & Control
Critical for SLT

Designed for control, auditability, and safety.

Agentic AI without guardrails is a liability. Every action is permissioned, every decision is logged, every high-risk step has a human in the loop.

Controls

  • Role-based permissions for actions
  • Human-in-the-loop for high-risk decisions
  • Approval workflows for critical steps

Risk Mitigation

  • Prompt injection protection
  • Tool access (least privilege)
  • Fail-safe escalation

Audit

  • Full decision logging
  • Explainability of actions
  • Compliance alignment
